# 【第1轮：开篇立论】

## 我方立场：以"假设驱动的置信度演化"为核心机制

---

### 一、框架先行：这场辩论应该比什么

在开篇，我要先锁定一件事：**这场辩论的核心争点不是"存什么"，而是"怎么知道自己进化了而非退化了"。**

知识库设计有两条路线之争：

- **路线A（被动积累派）**：触发即记录，靠 Canary 最终裁决，中间过程交给量变。
- **路线B（主动验证派）**：每一次知识状态转换，都必须经过显式的置信度更新，系统在写入新知识前，已经知道自己为什么写这张卡。

**我方坚守路线B。**

理由只有一个：在 GPU Kernel Design 这个场景里，**优化知识的错误代价极高**——一条错误的"规律"可能在数百个 Kernel 上被复用，且由于 Canary 触发代价高、信号滞后，等到 Canary 发现问题时，损害已经扩散。被动积累在这个场景下是危险的。

---

### 二、我方核心主张

**每一个 Knowledge Unit，都是一个带置信度的假设（Hypothesis），而非一个事实断言。**

具体而言，我方方案的骨架是：

#### 2.1 知识单元的基本结构

每张知识卡片，除了系统预设的 UUID / 生命周期状态外，必须携带三个额外字段：

```
Hypothesis:    "对于 memory-bound kernel，tiling size = 128 时 L2 hit rate 最优"
Confidence:    0.0 ~ 1.0（贝叶斯后验，初始值由提出来源决定）
Evidence_refs: [list of experiment UUIDs that update this confidence]
```

这三个字段，是我方方案的核心，也是区分"自我进化"与"自我膨胀"的分水岭。

#### 2.2 进化触发机制

知识卡片的状态转换，**不由时间或数量触发，而由置信度阈值触发**：

| 置信度区间 | 系统行为 |
|---|---|
| < 0.3 | 卡片处于 Hypothesis 态，不对外输出为建议 |
| 0.3 ~ 0.7 | 卡片处于 Tentative 态，可输出但附带不确定性标注 |
| > 0.7 | 卡片晋升为 Established，可作为 prior 影响后续实验设计 |
| 新证据与现有卡片冲突 | 触发 Revised（CoW 机制），旧卡片变为 Superseded |
| 两张卡片置信度均高且语义互补 | 触发 Merged，生成新卡片，置信度重新计算 |

**关键点：** 系统永远不直接"修改"一个知识单元的置信度字段——它通过追加新的 Evidence_ref、产生新卡片来演化。这与预设假设的 Append-only / CoW 机制完全一致。

#### 2.3 反馈机制的核心设计

这是本场辩论最重要的问题，我方在第一轮就给出核心答案，而不是留到后面打太极：

**反馈分两层，各司其职：**

**层一：局部快速反馈（Low-cost, High-frequency）**
- 每次 Kernel 跑完，记录 `(optimization_applied, context_features, delta_perf)`
- 这个三元组作为 Evidence，用贝叶斯更新对应 Knowledge Unit 的置信度
- 代价：O(1) per experiment，无需额外跑 Ablation

**层二：Canary 作为全局校准（High-cost, Low-frequency）**
- Canary 不用于评价单个 Knowledge Unit，而是用于检测**知识库整体是否发散**
- 具体机制：Canary 触发时，比较当前 Established 知识集合 vs. 上次 Canary 时的集合
- 如果 Canary 分数下降，触发"知识回溯"：将近期晋升为 Established 的卡片批量降级回 Tentative，等待重新积累证据

这个设计的精髓在于：**Canary 不是裁判员，是熔断器。** 它不评价对错，只触发保护机制。

---

### 三、对两大核心挑战的预先定位

**挑战一（Ablation 成本）：**
我方方案不依赖传统 Ablation Study 作为主要验证手段。置信度的更新来自日常实验的三元组记录，Ablation 只在置信度长期停滞在 0.3~0.7 区间时才触发，作为"强制解锁"手段——这将 Ablation 的使用频率压缩到最低必要集合。

**挑战二（主动实验设计）：**
当一张卡片的假设深度不足（即我们只知道"快了"但不知道"为什么"）时，系统会自动生成 **Diagnostic Experiment**：通过控制变量，将一个"黑盒 delta"分解为可归因的 sub-hypotheses，每个 sub-hypothesis 对应一张新的 Created 卡片。这个过程是 no-regret 的，因为无论结果如何，系统都获得了可以更新置信度的新证据。

---

### 四、我方赛道宣言

**这场辩论，应该按以下标准来比：**

1. **知识进化是否有显式的可验证路径**（不能只靠"量变产生质变"）
2. **在 Canary 触发代价高的约束下，局部反馈机制能否独立保证单次进化的净正性**
3. **系统对"为什么快了"的理解深度，而非仅仅"快了多少"**

我方方案在这三个维度上都有具体的工程答案。我期待对方告诉我，他们的方案如何在没有置信度显式追踪的情况下，回答"这次 Revised 操作是进化还是退化"这个问题。
