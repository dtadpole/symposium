# 【第1轮：开篇立论】

---

## 一、明确立场

我方核心主张：**GPU Kernel 自动优化系统的知识库自我进化，应以"结构化假设驱动"为核心范式，而非依赖被动积累与事后验证。**

具体而言：知识库的每一次进化操作，必须由一个显式的、可证伪的 Hypothesis 发起，而非由性能数字的被动变化触发。系统的核心能力不是"记录什么有效"，而是"理解为什么有效，并知道下一步该验证什么"。

这与另一种可能的立场——"以 Canary 为中心、累积足够变化后再验证"的被动进化模型——形成直接对立。

---

## 二、关键定义

**Knowledge Unit（KU）**：一个可证伪的优化假设卡片，包含：
- `hypothesis`：优化为何有效的结构性解释（不是"快了3%"，而是"减少了 bank conflict 因为…"）
- `applicability_condition`：在哪类 kernel / pattern 下适用
- `evidence_set`：支持或反驳该假设的实验记录
- `confidence_score ∈ [0,1]`：基于证据强度的量化置信度
- `lifecycle_state`：Created / Revised / Merged / Split / Superseded / Archived

**进化（Evolution）**：KU 的状态转换，当且仅当有新证据导致 `confidence_score` 或 `applicability_condition` 发生显著变化时触发，而非时间驱动或频率驱动。

**净进化（Net Evolution）**：一次状态转换操作后，系统在 held-out local benchmark 上的期望性能不下降，且知识结构的可解释性（measurable by hypothesis depth）不退化。

---

## 三、判断标准

本场辩论的胜负标准应为：**哪个方案能在资源受限条件下，以最少的无效实验代价，最大化知识库的"可迁移解释能力"**——即不只是在已见过的 kernel 上表现好，而是能将 KU 中的 hypothesis 迁移到新 kernel 并产生可预测的优化收益。

这个标准由三个维度构成：
1. **验证效率**：每单位计算资源带来的 confidence 提升
2. **解释深度**：KU 中 hypothesis 的可证伪程度与因果链完整性
3. **迁移能力**：KU 在 out-of-distribution kernel 上的预测准确率

---

## 四、两个核心论点

### 论点一：反馈机制必须在 KU 层面闭环，而非依赖 Canary 全局信号

Canary Query Set 是长期 ground truth，但它有两个根本性局限：
- 运行代价高，不可频繁触发
- 单次 KU 变化在 Canary 上几乎不可见（信号被淹没在噪声中）

因此，**Canary 不能作为 KU 级别进化的判断依据**，只能作为系统整体健康度的长期校验。

我方方案：每个 KU 维护一个轻量级的 **Local Verification Set（LVS）**——针对该 KU 的 `applicability_condition` 精心选取的 3-5 个 micro-benchmark，专门用于验证该 KU 的 hypothesis 是否成立。

进化触发逻辑：
```
if Δconfidence(KU) > θ_trigger (e.g., ±0.15):
    → 触发状态转换评估
    → 在 LVS 上运行 targeted ablation
    → 根据结果更新 evidence_set 和 lifecycle_state
```

这样将验证成本从 O(全局 Canary) 降低到 O(单个 KU 的 LVS 规模)，且每次验证的信号都直接指向该 KU 的 hypothesis。

---

### 论点二：系统必须具备主动实验设计能力，且以"no-regret 实验优先"为排序原则

当 KU 的 hypothesis 深度不足时（即只知道"快了"，不知道"为什么快"），系统不应等待，而应主动生成 sub-hypothesis 并设计验证实验。

**触发条件**（何时启动主动实验）：
```
if KU.hypothesis_depth < δ_min  // hypothesis 缺乏因果链
OR KU.confidence_score ∈ (0.35, 0.65)  // 证据高度不确定
→ 触发 Hypothesis Deepening Protocol
```

**实验优先级排序**（如何分配资源）：

定义每个候选实验 $e_i$ 的优先级得分：

$$\text{Priority}(e_i) = \frac{\mathbb{E}[\Delta \text{confidence}(e_i)] \cdot \text{transfer\_potential}(e_i)}{\text{cost}(e_i)}$$

其中：
- $\mathbb{E}[\Delta \text{confidence}]$：预期置信度提升（基于历史相似实验估计）
- $\text{transfer\_potential}$：该实验结论能被复用到多少其他 KU
- $\text{cost}$：计算资源消耗的归一化估计

**No-regret 实验的定义**：满足以下条件的实验：
- $\text{cost}(e_i) < \epsilon_{budget}$（预算阈值以下）
- 无论结果如何（validate 或 falsify），都能更新至少一个 KU 的 `evidence_set`
- 即使 falsify，也产生有价值的 negative evidence（可触发 Superseded 或 Archived 转换）

高成本、收益不确定的实验（如全 kernel 重写级别的验证）必须满足 $\text{Priority}(e_i) > \tau_{high\_cost}$ 才可进入执行队列，且需锁定预算配额。

## 五、核心焦点

本场真正的争议在于以下两点：

**焦点一：进化的驱动源是"假设驱动"还是"信号驱动"？**
我方认为必须先有 hypothesis 才能设计有效实验；对方可能认为只要累积足够多的性能信号，系统自然涌现出有效的 KU。这是根本性的架构分歧。

**焦点二：在资源极度受限时，"验证覆盖"与"假设深度"应如何权衡？**
