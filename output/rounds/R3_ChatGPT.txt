一、回应对方质询（防守）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

**对方核心质疑（重建）：**
对方很可能在两处发起攻击：
1. 我方的"置信度评分"本质上是主观的，缺乏真实 ground truth 锚定；
2. 我方的分层验证在 Ablation 代价高昂时会退化为"不做验证"，与直觉驱动无本质区别。

**正面回应：**

**关于置信度的客观性问题：**

我方从未声称置信度是主观打分。置信度 C(k) 是以下三个可测量信号的函数：

> C(k) = f( 复现率 R, 效果一致性 δ_consistency, 依赖隔离度 I )

- **R（复现率）**：同一优化策略在不同 kernel 形态（不同 M/N/K、不同 memory layout）下被独立触发并验证成功的次数，纯计数，客观；
- **δ_consistency**：多次实验中加速比 δ 的方差，方差小则置信度高，统计量，客观；
- **I（依赖隔离度）**：该 Knowledge Unit 被激活时，同时激活的其他 KU 数量的倒数——共激活越少，归因越干净，可计算，客观。

这三个维度没有一个是靠人工感觉打分的。对方若质疑置信度主观，请指出这三个指标哪一个无法在系统中自动采集。

**关于"不做验证"的退化风险：**

我方的分层验证设计本身就是为了应对高代价场景。关键在于：**验证层级的选择不是随机的，而是由假设的影响范围决定的**。

- 影响范围小（单 kernel、单 op 类型）→ 微基准验证，代价极低，必做；
- 影响范围中（同 op 类型跨 shape）→ Mini-Canary（Canary 的 10% 子集），代价中等，按 KU 批次触发；
- 影响范围大（跨 op 类型的全局规律）→ 完整 Canary，代价高，严格限频。

这不是"选择不做验证"，而是**让验证代价与假设影响范围成比例**。低影响的假设绝不消耗高代价的 Canary 资源，高影响的假设绝不跳过验证直接写入知识库。

对方方案（若是纯 Canary 驱动或纯经验积累）的真正风险是：大量中等影响的假设缺乏验证通道，只能等 Canary——这才是真正的验证真空。

二、主动攻击对方漏洞
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

**攻击点一：对方的"无效实验"处理机制是黑洞**

对方在第2轮中（基于前两轮脉络推断）对 Ablation 的处理是：做实验，如果没有 delta，就记录"无效"然后跳过。

这个设计有一个致命漏洞：**"无效"本身包含两种完全不同的信息**，对方没有区分：

- **类型A：假设本身错误**——这个优化方向对这类 kernel 无效，应该写入知识库作为负面知识（"在 memory-bound kernel 上，增加 register tiling 无益"）；
- **类型B：实验设计不足以检测**——delta 太小是因为测试的 shape 不覆盖假设的适用域，换个 shape 可能有显著 delta。

如果对方不区分这两类，"无效"就会被统一丢弃，导致：
1. 类型A的负面知识丢失 → 系统重复探索已知无效方向，浪费实验预算；
2. 类型B的有效假设被误杀 → 知识库欠拟合，进化停滞。

**我方如何处理：** 每次实验结束，不仅记录结果，还记录**实验的覆盖域**（哪些 shape、哪些硬件配置被测试）。"无 delta"的结论只在该覆盖域内有效，并以 `NullResult(domain=..., confidence=...)` 的格式写入知识库。这使负面知识也成为可检索、可复用的结构化资产。

---

**攻击点二：对方的假设生成缺乏"可证伪性"约束**

如果对方的假设生成是由 LLM 或经验规则自由生成的，那么生成出来的假设可能根本无法被当前实验条件所验证。

例如："shared memory bank conflict 导致该 kernel 慢" → 这个假设需要特定的 profiling 工具（如 Nsight）和特定的实验设计（系统性地改变 padding），不是随便跑一个 benchmark 就能验证的。

**可证伪性约束**是假设质量的门槛，不是可选项。我方要求每个假设在生成时必须附带：

> `验证协议 V`：明确指定验证该假设所需的最小实验集合，包括变量、控制条件、成功/失败判定标准。

没有验证协议的假设不进入实验队列。对方若跳过这一步，实验队列会被大量"无法验证"的假设堵塞，消耗资源却无法产出置信度提升。
