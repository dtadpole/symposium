## 争点一：效用度量的可操作性——我方的最终论证

**对方的质疑**：效用 U(k) 依赖事前预测，而预测模型在分布偏移时失灵，导致净进化保证名存实亡。

**我方的完整回应**：

对方混淆了两件事：**效用估计的精确性**与**效用估计的方向性**。

贝叶斯框架从未要求精确预测 U(k) 的绝对值。它要求的只是：当 Alpha(k) 持续累积正向证据时，后验分布向右移动；当 Beta(k) 累积负向证据时，向左移动。这个**单调性**在分布偏移下仍然成立——因为新的查询会产生新的观测，观测本身就是更新先验的信号，而不是依赖旧模型的预测。

具体来说，在分布偏移场景下：

- 旧卡片在新查询上 Score(t | KB) 下降 → Beta(k) 增加 → LCB(k) 下降 → 触发 Archive 或 Revised
- 新领域的新卡片被引用后产生正向反馈 → Alpha(k) 增加 → 卡片权重上升

**这不是预测，这是在线学习（online learning）**。预测门槛 ΔUD_predicted > 0 是保守约束，不是必要条件——即使预测不准，负向演化会通过实际观测的 ΔUD_actual 触发回滚。对方把"预测不准"等同于"机制失效"，是误解了双重保障结构：事前预测阻断明显错误操作，事后观测纠正漏网错误操作。两道门都需要穿透，机制才真正失效。

**数学上**：设预测误差为 ε，只要 |ε| < |ΔUD_true|，预测门槛仍然有区分力。在分布偏移时，大的效用变化（正负）恰好使得 |ΔUD_true| 变大，门槛的区分力反而更强，而非更弱。

## 争点二：R(k) 的冷启动悖论——我方的完整修复方案

**对方的质疑**：新卡片因引用频率低导致 R(k) 低，在资源压缩时被优先淘汰，形成马太效应。

**我方承认这是真实问题，并给出具体解法**：

冷启动问题的根源是 R(k) 完全依赖历史引用频率，忽略了**年龄信息**。修复方案如下：

将 R(k) 分解为两个分量：

```
R(k) = λ · R_usage(k) + (1-λ) · R_novelty(k)
```

其中：
- R_usage(k) = 基于历史引用频率的衰减权重（原设计）
- R_novelty(k) = max(0, 1 - age(k) / T_grace)，在宽限期 T_grace 内给新卡片一个线性递减的保护权重
- λ ∈ [0,1] 是混合系数，可设为 λ = min(1, age(k) / T_grace)——即随年龄线性从 0 增长到 1

**效果**：新卡片在 T_grace 期内（比如前30次查询）被完全保护，不参与 Archive 竞争；T_grace 过后，完全切换到使用频率驱动。马太效应被切断，因为旧卡片不再能在"新卡片还没有机会积累引用"的窗口期内将其挤出。

**更重要的是**：对方说"查询机制偏向旧卡片"——这个问题的根源在检索层，而不在效用评分层。效用评分的修复不能替代检索层的去偏，但也不应该被检索层的偏差拖累。两层分开修，是正确的工程分层原则。

## 争点三：Merged/Split 后的回滚语义——我方的精确定义

**对方的质疑**：回滚在 Merged/Split 操作后语义模糊，M 和 k1/k2 同时存在时权威版本不清晰。

**我方给出精确的一致性语义**：

CoW（Copy-on-Write）架构下，所有卡片都有版本号和状态位。定义如下状态机：

```
状态集合：{Active, Superseded, Archived}
查询层规则：只有 Active 状态的卡片参与检索
```

Merged 操作的执行步骤：
1. 生成新卡片 M，状态 = Active，记录 provenance = {k1, k2}
2. k1、k2 状态 → Superseded（不删除，仍存储）
3. 观测窗口期（例如后续 50 次查询）内计算 ΔUD_actual

回滚触发条件（ΔUD_actual < -δ）：
1. M 状态 → Archived
2. k1、k2 状态 → Active（从 Superseded 恢复）
3. 系统重新索引，查询层立刻生效

**这里没有歧义**：查询层始终只看 Active 状态，权威版本在任何时刻都是唯一确定的。Superseded 的卡片存在于存储层但不参与检索，回滚只是状态位的切换，不是内容的重新生成。

对方说"M 和 k1/k2 同时存在"——是的，它们同时存在于**存储层**，但不会同时存在于**查询层**。这个分层是 CoW 架构的基本设计，不是漏洞。

**Split 操作的回滚同理**：原卡片 k 状态 → Superseded，生成 k_a、k_b 状态 → Active。回滚时 k_a、k_b → Archived，k → Active。语义同样清晰。
